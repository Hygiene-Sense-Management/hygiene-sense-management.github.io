Execution Scope

 Top  Previous  Next

When a device filter is executed, it is always executed in a specific scope. This execution scope defines the base set of devices to which the filter will be applied. It also is basis for the permission check that is performed before the execution of the device filter. I.e. the user executing the device filter must have the permission to see all devices in this scope.

The UIs available in the HsM system usually make the current scope very explicit (e.g. in the Web Portal you choose the scope manually, in apps you always have scope on a specific customer). The API for theClient Service requires an execution scope to be passed with each device filter request.

Depending on the execution scope the Client Service will dynamically modify the device filter to make sure that the results will only include devices in the given scope.

Global scope

The global execution scope includes all devices managed by the Client Service. Device filters executed in global scope are not modified dynamically.

Organization scope

Includes all devices of all customers within an organization.

If a customer has sub-ordinate chain and bill-to customers those customers will always be in the same organization. When executing a query in an organization scope the devices of any sub-ordinate customer of any customer in that organization are part of the base set of devices to which the filter is applied.

Customer scope

Includes all devices of the customer or one of its chain, or bill-to customers.

Customers can have associated chain customers and associated bill-to customers. When executing a device filter in a customer scope the devices of these sub-ordinate customers will also be part of the base set of devices on which the filter is executed.

Note chain to and bill customer are hierarchical, e.g. chain customers can have sub-ordinate bill-to customers etc.
