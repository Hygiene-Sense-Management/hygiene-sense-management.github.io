Creating a Windows Service

 Top 

While there are many ways to register a program as a Windows Service, we recommend the following approach with PowerShell. Note you need to run PowerShell with administrative privileges to modify Windows Services.

Creating a Windows Service for the Digital Twin Registration Agent can be done with the New-Service PowerShell cmdlet. For full documentation of this cmdlet type Get-Help New-Service -Online  in PowerShell.

Note that multiple windows service can be created based on the same installation↲↲↲ of the Digital Twin Registration Agent. However make sure that these service use different Environments and therefore different configuration. The Digital Twin Registration Agent will not allow two instances to run with the same configuration.

General Syntax of the New-Service cmdlet:

New-Service 

 [-Name] <String> 

 [-BinaryPathName] <String> 

 [-DisplayName <String>] 

 [-Description <String>] 

 [-SecurityDescriptorSddl <String>] 

 [-StartupType <ServiceStartupType>] 

 [-Credential <PSCredential>] 

 [-DependsOn <String[]>] 

 [-WhatIf] 

 [-Confirm] 

 [<CommonParameters>]

 

Suggestions for the parameters to use:

Parameter

Notes

-Name

It is recommended to put the environment name in the service name so that multiple service can be installed, e.g.: DTRA_<EnvironmentName>

The name cannot be null or have zero length. Its maximum size is 256 characters. It also cannot contain forward or backward slashes, '/' or '\', or characters from the ASCII character set with value less than decimal value 32.

-BinaryPathName

The full path to the executable of the Digital Twin Registration Agent including the command line switch for the Environment. Environment name should be consistent with the -Name parameter.

'"<FullPathToExe" "-Environment=<EnvironmentName>"'

-StartupType  

Recommended: Automatic

 

Example

New-Service `

 -Name "DRTA_Production" `

 -BinaryPathName '"<path>\Hsm.DigitalTwinRegistrationAgent.exe --Environment=Production" `

 -DisplayName "HsM Digital Twin Registration Agent - Production" `

 -Description "Registeres digital twins with the HsM Digital Twin Service" `

 -StartupType "Automatic"
