Logging

 Top 

For monitoring the Digital Twin Registration Agent the program logs via the nlog framework and therefore allows for versatile routing of log messages.

Log messages have associated log levels that can be used to route log message. The Digital Twin Registration Agent uses log levels in following way:

Log level

Notes

Fatal

Exceptions that prevent the Digital Twin Registration Agent from operating. Io Exceptions, Network down etc. - must be actively monitored.

Error

Exceptions that hinder registration of devices. Must be actively monitored

Warning

Require Weekly checks by administrators

Info

Detailed Protocol of operation. Can be used by administrators to debug behavior

Debug

Only for developer debugging

Trace

Only for developer debugging
