Recording of data link status

 Top  Previous  Next

The data link status of a HsM device is recorded as resource instance values for each device. I.e. the data link status of each HsM device is recorded as time-series of data.

Data-link status change (Non-gateway devices and stationary gateways)

Whenever a change in the data-link status of a device is detected by the HsM data link monitoring the new status after the status change is recorded as a resource instance value and stored with the device. Recorded status can be:

•Online

•Offline - Device link lost (will not be used for stationary gateways)

•Offline - Gateway link lost

Due to technical reasons the timestamp of this change in not always precise as the change of the data link status might be detected with a certain delay.

Last used stationary gateway (Non-gateway devices only)

For non-gateway devices the last used stationary gateway, i.e. the stationary gateway that was last used by the non-gateway device to connect to the  HsM backend) is stored.

Last-detected-online (Non-gateway devices and stationary gateways)

The timestamp when a data-link status of the device was last Online. This is used to for differentiating between temporarily offline and offline.
