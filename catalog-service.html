Catalog Service

The HsM Catalog Service provides meta data about all HsM device types. This meta data is used in many areas of HsM to dynamically adapt to new device types or changes to existing device types.The HsM Catalog Service is part of the HsM Neutral back-end infrastructure. The meta data served by the Catalog Service is solely modified by Hagleitner. Read-only access to the meta data in the Catalog Service is however not restricted. The Catalog Service does not provide a user interface but is intended to be used by other software systems.

The meta data managed by the Catalog Service contains (not necessarily complete list):

•Unique device type IDs: Each HsM device type has a unique ID (a global unique identifier), the catalog service is the place where these IDs are assigned on registration of new device types.

•Structural models of device types: Including the device’s component placeholder and compound member placeholders.

•Definition of the resource instances (sensors and settings) exposed by firmware components within the device type. This includes a definition of the resource instances data type, validation rules, display hints etc.

•Naming of the device type, its components, etc. in multiple languages
