Latest Resource instance value

 Top  Previous  Next

Filters devices by their latest reported resource instance values.

A resource instance is specified either by device type id, resource id and resource instance number or by resource instance equivalent and device types.

Restrictions

If a resource instance with a visibility level not available to the executing user is specified no device will pass the filter.

Settings

Setting

Description

Resource specification

This setting defines the queried resource instance. The actually queried resource instance can be given either as specific resource for a specific device type or as a resource instance equivalent for a set of selected device types. Possible settings are therefore:

•Specific resource instance

•Resource instance equivalent

Settings for Resource specification = Resource instance equivalent

Resource instance equivalent

A resource instance equivalent (as specified by the Catalog Service)

Device types

A list of device types that shall be included in the filter. Device types that do not have a resource instance equivalent

Settings for Resource specification = Specific resource instance

Device type

The device type included in the filter

Resource type

The resource type

Resource instance no

The resource instance number.

Comparison settings

Setting

Description

Comparison operator

Depending on the data type used by the selected resource type comparison operators are allowed:

•Numeric with single comparison value: >, <, >=, <=, =, !=

•Numeric with two comparison values: Between (>= Comparison value 1 and <= Comparison Value 2)

•String: Starts with, Equals

Comparison value 1

The comparison value used for operators with a single comparison value

Comparison value 2

The 2nd comparison value used for operators with two comparison values
