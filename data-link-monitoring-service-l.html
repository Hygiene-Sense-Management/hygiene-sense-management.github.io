Data link monitoring service logic

 Top  Previous  Next

The data link monitoring service records the result of monitoring data links between non-gateway devices and stationary gateways and the results of monitoring the data links between stationary gateways and the HsM backend. It keeps the current online status of these devices and in case of changes to the status will forward the changed status to the HsM IoT Data Storage as resource instance values.

•Changes to a device status will be persisted as resource instance values. Consecutive reporting of the same status will not be persisted in the associated resource instance values.

•Persisting as resource instance values is done via IoT hub (as fake resource instance values)

•When a stationary gateway disconnection is detected all non-gateway devices that used this stationary gateway previously will be set to state offline (Gateway not reachable) if the device has not been reported online with another stationary gateway within a grace period of xx minutes.

•Devices for which no update has been received for the last 24 hours will be set to Offline (Device not reachable).
