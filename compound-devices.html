Compound devices

 Top  Previous  Next

The Digital Twin service supports compound device types in that way that it records the current relationship between devices and also the historical relationships between devices.

 

Terminology:

A device that has another device built-in is called a compound device

A device that is part of a parent device is called a compound member device

 

Getting the device id from a component id

When querying the digital twin service for the device by the identifier of a component this query can be issued either for the component’s device or for the component’s root device.

 

Getting the device ID by identifier of a component

When evaluating the device id the digital twin service will first search for the given component id and will check if this component is registered with some device. If yes it will return the ID of that device.

 

Getting the root device by identifier of a component

When evaluating the root device id the digital twin service will first search for the given component id and will check if this component is registered with some device. Then the digital twin service will check if the device of the component is a contained device, if so it will return the id of the compound device, if not the device id is returned.

 

Combining devices

Compound devices can be either created by the Digital Twin registration agent when the device is first registered or can be created through the HsM Client Service (usually via Xibu App or Web Frontend)

 

The flow for combining devices through one of the frontends would be:

 

Select parent device's combined device type placeholder. E.g. go to the Multifold towel, select the MLE placeholder. As device type placeholders are typed the software knows which types of devices can be mapped to this placeholder.

 

Select a device of that type (usually by scanning an identifiable component, or by selecting from a list of devices associated with the same customer)

 

The combinable device will then be mapped to the compound device in the Digital Twin Service, any other relation of the combinable device to a compound device will be removed.

 

Changes in the digital Twin Service will be immediately synced to the HsM Client Service.
