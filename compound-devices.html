Compound devices

 Top  Previous  Next

The Digital Twin Service supports compound device types. It records the current relationship between devices and keeps a history of relationships between devices.

Terminology:

A device that has another device built-in is called a compound device.

A device that is part of a parent device is called a compound member device.

 

Getting the device id from a component id

When querying the digital twin service for the device by the identifier of a component this query can be issued either for the component’s device or for the component’s root device.

 

Getting the device ID by identifier of a component

When evaluating the device id the digital twin service will first search for the given component id and will check if this component is registered with some device. If yes it will return the ID of that device.

 

Getting the root device by identifier of a component

When evaluating the root device id the digital twin service will first search for the given component id and will check if this component is registered with some device. Then the digital twin service will check if the device of the component is a contained device, if so it will return the id of the compound device, if not the device id is returned.

 

Combining devices

Compound devices are created by instances of the Client Service. I.e. the relationship between a compound device and a compound member device is created by a user through some user interface and then the Client Service will forward this relationship information to the digital twin service.
