Handling of invalid device filters

 Top  Previous  Next

Device filters may be invalid due to various reasons (see below).The Client Service will handle invalid device filters gracefully, i.e. the execution of the filter will succeed even for invalid filters, in that case the result of the execution might however not be exactly what the user intended.

Handling of incomplete expression tree

Incomplete expression trees are handled by the Client Service by applying following checks and corrections (in the order as listed below)

•Missing leave nodes beneath a logical expression node: If a logical expression node does not have any children, the client service will remove this node.

•Missing root node: If the device filter does not have a root node, the client service will dynamically modify the device filter to an expression with a root AND note and a single Falsification term. The result set of such a device filter will therefore always be empty.

Invalid filter term

Each device filter term type defines its own validity constraints. See the individual device filter term type documentation for more details.

Contradicting terms

The combination of terms in a device filter may result in contradictions. This is not checked by the Client Service. If the combination of nodes results in a logical contradiction the result set of devices will therefore always be empty.
