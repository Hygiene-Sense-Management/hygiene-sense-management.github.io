Data Link Monitoring Service

HsM stationary gateways are used to connect non-gateway devices to the HsM backend permanently and therefore enable a near-real-time uplink-communication (from devices to the HsM backend) and near real-time downlink-communication (from the HsM backend to devices).

When non-gateway devices are using stationary gateways to communicate with the HsM backend, the devices do so by utilizing a Bluetooth connection to a stationary gateway. This stationary gateway in turn uses a MQTT over TCP to relay this data to or from the HsM backend via the Internet. While this communication sequence usually is very stable things can still go wrong due to many reasons; Bluetooth connections can break, internet connection may be down, devices may run out of energy, stationary gateways may be down etc.

HsM data link monitoring continuously analyzes the data links and therefore provides information about the data link status of devices (both stationary gateways, and non-gateway devices) as well as information for a historical analysis of data link quality.
