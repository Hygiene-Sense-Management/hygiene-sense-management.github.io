Prerequisites

•.NET Core 3.1 runtime: The Digital Twin Registration Agent is a .NET Core 3.1.10 application and therefore the .NET Core 3.1 run time must be installed.

•ASP.NET Core 3.1.10 run time : The Digital Twin Registration Agent uses ASP.NET Core components, therefore the .NET Core 3.1.10 run time must be installed.

•Network connection to the Provider DB: The Digital Twin Registration Agent reads data from the Provider DB via ADO.NET. Therefore the appropriate network and firewall configuration must be set up.

•Network connection to the Digital Twin Service The Digital Twin Registration Agent accesses the REST API of the Digital Twin Service. Therefore the appropriate network and firewall configuration must be setup.

•Network connection Digital Twin Service The Digital Twin Registration Agent accesses the REST API of the Digital Twin Service. Therefore the appropriate network and firewall configuration must be setup.

Installation of program files

The Digital Twin Registration Agent is delivered as a single ZIP compressed file, the contents of that ZIP file must be extracted to the chosen program directory, usually C:\Program Files\HsM\Digital Twin Registration Agent\.

The Digital Twin Registration Agent can be run without administrative privileges.

Note: the Digital Twin Registration Agent can only be run after the configuration is done.
